## üöÄ How to Upload (Step-by-Step)

### Option 1: Replace Files One by One

**For each file:**
1. Go to your GitHub repository
2. If the file already exists (like `_config.yml`):
   - Click on the filename
   - Click the pencil icon (edit)
   - Delete all content
   - Paste new content from the downloaded file
   - Click "Commit changes"
3. If it's a new file:
   - Click "Add file" ‚Üí "Create new file"
   - Name it exactly as shown above
   - Paste the content
   - Click "Commit changes"

### Option 2: Upload Multiple Files at Once

1. Download all files from the outputs folder
2. Go to your GitHub repository
3. Click "Add file" ‚Üí "Upload files"
4. Drag and drop all the files
5. Click "Commit changes"

**Important:** For the `assets/` folder:
- Create folder structure: `assets/css/` and `assets/images/`
- Upload `custom.css` to `assets/css/`
- Upload `logo.svg` to `assets/images/`

## üîß Troubleshooting

**Site not updating?**
- Wait 5 minutes (sometimes GitHub is slow)
- Check Settings ‚Üí Pages to see build status
- Clear browser cache (Ctrl+F5 / Cmd+Shift+R)

**Styling looks broken?**
- Make sure `custom.css` is in `assets/css/` folder
- Check file uploaded correctly

**Navigation missing pages?**
- Make sure all .md files are in root directory
- Check they have proper front matter (---layout: page---)
